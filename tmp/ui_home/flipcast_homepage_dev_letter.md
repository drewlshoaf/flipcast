# Developer Letter

Team,

We are updating the Flipcast home page so it stops feeling like a studio surface and starts feeling like a content-rich media discovery experience.

The current direction is a **home page built around abundance, curiosity, and instant start behavior**. The user should land on the page and immediately feel that Flipcast is full of things to explore. It should feel playful, bright, modern, and consumer-friendly. It should not feel like a dashboard, settings page, or editor-first workflow.

The new home page concept is centered on a **large pink play button** with a surrounding field of many small topic bubbles. Those bubbles represent discoverable prompts, questions, and cultural topics. The effect should suggest that there are hundreds of starting points available.

There is one important layout correction from the latest review: the **headline and supporting paragraph need a protected read zone**. The floating topic cells should not crowd or visually compete with the central messaging. The copy block must remain easy to scan, calm, and intentional, while the topic cloud lives around it.

The style guide we established earlier is the source of truth. Where the existing preview diverges from it, the style guide wins.

The product principle to keep in mind is this:

**Flipcast is not a settings page. It is a media creation experience.**

The home page should constantly imply:
- you can start quickly
- you do not need to get everything perfect
- you can iterate after generation
- making a Flipcast is supposed to be fun

Please implement the home page using the attached concept direction and the specifications below.
